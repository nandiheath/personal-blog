樂園復活計劃 - 0

準備開始時就已經有一個晴天霹靂
上年寫的LS2READER的源碼弄丟了
應該是之前洗機的時候把在C:的源碼刪了…

印象中我好像寫了好幾天
而且殺死了好多腦細胞的 Orz

幸好當時參考某大大寫的SA技術文還在…
不然整個計劃還沒開始就ＧＧ了　(Orz…)

所以先來計劃下目前的幾個大方向

  1. 參考大大的源碼　把之前弄丟的exporter寫出來 (c# + VS)
  2. 研究cocos2dx 直接讀取bin檔的可能性 (c++ + xcode?)
    2.1 如失敗，研究其他rendering method? (圖片太多, 大概６萬張bmp)
    2.2 或者其他game engine? (儘量用開源並且支持多平台 web/win/ios/and)
  3. 選一個server side framework
    3.1 目標nodejs?
    3.2 smartfox 可以備選
    3.3 新挑戰可以選python/golang
  4. 想像一下要加新圖檔的流程
    4.1 bin file 難度高嗎?
